STAGE 01
Scene Ingestion
RGB-D panoramas from the Structured3D dataset are read. Depth maps, camera poses, and semantic annotations are extracted from each scene folder.
STAGE 02
3D → Bird's Eye View
Panoramic depth is unprojected into a 3D world-coordinate point cloud. Points are projected top-down into a fixed 512×512 BEV grid — 8 semantic channels, view-invariant.
STAGE 03
RoomToFP v7 Inference
The hybrid CNN-Transformer architecture processes the BEV tensor. The polygon decoder predicts room instances with corners. The mask decoder predicts walls, floors, and openings.
STAGE 04
Post-Processing
Low-confidence rooms are filtered. Polygons are cleaned and snapped to axis-aligned geometry. Wall and opening masks are thresholded and combined into the final layout.
STAGE 05
Floorplan Output
A vector SVG floorplan is rendered — colour-coded rooms with labels, wall line segments, and opening markers for doors and windows. Also exported as PNG.
Core Technology

Inside RoomToFP v7

Our Final AI Architecture

max_rooms = 20

Learnable room queries, one per slot. Hungarian matching assigns each to a ground-truth room — order-invariant training.

max_corners = 16

Up to 16 polygon corners per room. Cyclic alignment ensures any valid vertex ordering receives the correct loss signal.

embed_dim = 256

Shared feature space across polygon and mask branches. Both learn from the same BEV representation simultaneously.

Research Journey

Seven Versions.
One Final Model.

Each version addressed a specific failure mode from the one before. The result is not just an architecture — it's a map of what floorplan generation actually requires.

Version 01
V1
Raster Baseline
Simple CNN encoder-decoder. Proved the pipeline could learn from Structured3D. No room-level reasoning whatsoever.
Version 02
V2
U-Net Refinement
Skip connections for spatial detail preservation. Improved wall edge accuracy. Still produced only pixel masks, no room polygons.
Version 03
V3
Multi-Channel BEV
Introduced 8-channel BEV input with separate wall, floor, opening, and furniture channels. Richer spatial evidence for the model.
Version 04
V4
First Polygons
Polygon prediction introduced. Rooms became geometric shapes rather than pixel blobs. Unstable due to vertex ordering ambiguity.
Version 05
V5
Hungarian + Cyclic
Hungarian matching for room-to-GT assignment. Cyclic alignment for polygon vertex ordering. Training became significantly more stable.
Version 06
V6
Hybrid Multi-Task
Polygon branch + raster mask branch sharing the same CNN encoder. Both complementary signals improved interpretability.
Version 07 ★ FINAL
V7
Best Balance
All strongest components combined. Selected for inference stability and output quality.

// KEY FINDING

V8's failure proved the bottleneck is not architectural complexity. It's geometric supervision, data scale, and polygon regularisation. V7 was selected not for being latest — but for being most stable and usable.

Detection Capabilities

What VisionScape Detects

🏠
Room Instances
Each room is detected as a separate instance with its own polygon boundary, class label, and confidence score. Up to 20 rooms per scene.
⬛
Wall Boundaries
Dense wall masks predicted at 512×512 resolution, then vectorised into line segments extracted from room polygon edges.
🚪
Door & Window Openings
Opening masks fused with BEV opening channel. Connected components projected onto nearest wall segment for architectural placement.
🏷️
Room Type Labels
Each predicted room is classified — bedroom, bathroom, living room, kitchen, balcony, study, and more. Trained with cross-entropy classification loss.
📐
Polygon Geometry
Rooms represented as ordered sequences of up to 16 corner coordinates. Iteratively refined across 3 decoder layers in normalised BEV space.
📄
SVG + PNG Export
Output as a colour-coded vector SVG with room labels and wall markers, plus a raster PNG preview — ready for downstream use.

What We Learned.
What Comes Next.

// CURRENT LIMITATIONS

01
Polygon Precision
The model identifies room locations and approximate shapes, but predicts clean architectural polygon boundaries inconsistently. Corners are sometimes misplaced, edges sometimes irregular.
02
Opening Detection
Doors and windows are small, sparse, and visually similar to wall gaps in the BEV. The model relies on raster mask post-processing rather than clean polygon prediction.
03
Layout Consistency
Combining multiple room predictions into a single coherent floorplan is harder than predicting rooms independently. Some outputs show overlaps or misalignments.
04
Post-Processing Dependency
The neural network alone produces raw predictions that are not yet clean enough for direct use. Filtering, cleaning, and thresholding stages are needed for usable output.

// FUTURE DIRECTIONS

🔷
Geometry-Aware Losses
Topology-aware and Manhattan angle losses to enforce rectilinear room geometry.
🔗
Polygon-Mask Consistency
Better alignment between polygon predictions and dense mask supervision for cleaner joint output.
🚪
Better Opening Prediction
Dedicated opening supervision and graph-based reasoning for reliable door/window placement.
🌍
Real-World Datasets
Training beyond synthetic Structured3D — scanned indoor datasets for real-world generalisation.
⚡
More Robust Inference
Reduce post-processing dependency through stronger model geometry outputs and improved decoders.
